🏷️ Variables

A variable is a labelled box that holds a value. Every Python program uses them.

📦
Think of it like a lunchbox Your lunchbox has a label on it (your name) and something inside (your sandwich). A variable is exactly the same — it has a name and holds a value inside.
HOW TO CREATE A VARIABLE
name = "Jordan" # the box is called "name" and holds "Jordan" age = 12 # the box is called "age" and holds 12 score = 0 # the box is called "score" and holds 0 # The = sign means STORE this value. It does NOT mean "equals" like in maths.

⚠️ 3 naming rules — break ANY of them and Python crashes

Rule 1: Only letters, numbers, and underscore _
Rule 2: Cannot START with a number → 3score ✗ but score3 ✓
Rule 3: No spaces → my score ✗ but my_score ✓

💡
Bonus tip: Use names that describe what's inside! player_name is a good name. x is bad — what does "x" mean?
👨‍🏫
Discussion: Ask the class — "If I wrote my-score with a hyphen, would Python accept it?" (No — hyphens are not allowed, Python thinks it's subtraction!)

🎨 Data Types

Every value in Python has a type. The type tells Python what it's allowed to do with it.

🧃
Think of it like different containers You wouldn't put soup in a paper bag or a letter in a bottle. Python is the same — different types of data need to be stored differently. Text goes in quotes. Numbers don't. True/False is for yes/no questions.

str — String (text)

Always inside quotes: "Hello" or 'Hello'
Even "7" is a string — it's text, not a number!

int — Integer (whole number)

Numbers WITHOUT a decimal point: 42, 0, -5
NO quotes! You can do real maths with these.

float — Decimal number

Numbers WITH a decimal point: 3.14, 1.5, 9.99
Division / always gives a float.

bool — Boolean (yes/no)

Only two values: True or False
ALWAYS capital T and F. Used for decisions.

🔑
The #1 rule to remember: "7" in quotes = string (text). 7 without quotes = integer (number). Python treats them COMPLETELY differently. "5" * 3 gives "555" but 5 * 3 gives 15.
👨‍🏫
Ask the class: "Why does Python need to know the type? What would happen if it didn't know?" (Answer: it wouldn't know if + means add numbers or join text together.)

📢 print()

print() shows something on the screen. It's how your program talks to the user.

📣
Think of it like a loudspeaker print() is Python's loudspeaker. Whatever you put inside the brackets gets announced on screen. No print(), no output — your program stays silent.

4 rules of print()

# Rule 1: Strings go inside quotes print("Hello!") # → Hello! # Rule 2: Variables go WITHOUT quotes name = "Jordan" print(name) # → Jordan (the VALUE inside the box) print("name") # → name (just the WORD "name") # Rule 3: Use comma to print multiple things print("Name:", name, "Age:", 12) # → Name: Jordan Age: 12 # Rule 4: Cannot join string + integer with + age = 12 print("I am " + age) # ✗ ERROR! Use str(age) or comma print("I am", age) # ✓ Works! → I am 12
🔑
Key difference: print("name") prints the WORD "name". print(name) prints whatever VALUE is stored in the variable called name.
👨‍🏫
Live demo: Show both print("name") and print(name) side by side in Thonny. Ask: "Why are the outputs different?" This is the most common confusion.

🎤 input()

input() lets your program listen to the user. It pauses and waits for them to type something.

🎙️
Think of it like a microphone print() is the loudspeaker (talking TO the user). input() is the microphone (listening FROM the user). Together they make a conversation.

4 rules of input()

# Rule 1: input() pauses and waits name = input("What is your name? ") # Python shows the question, then WAITS for the user to type # Rule 2: Always store in a variable name = input("Name? ") # ✓ stored in name input("Name? ") # ✗ answer is LOST forever # Rule 3: input() ALWAYS gives you a STRING age = input("Age? ") # user types 12 → stored as "12" (text!) # Rule 4: Use int() to convert to a number age = int(input("Age? ")) # now age is 12 (number!) ✓
⚠️
The big trap: input() ALWAYS gives you a string — even if the user types a number. If the user types 12, Python stores "12" (text). Wrap with int() to get a real number.
👨‍🏫
Live demo: In Thonny, type age = input("Age? ") then print(type(age)). Show them it says <class 'str'> even though they typed a number.
